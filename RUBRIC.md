# Evaluation Rubric: Slot Machine App

## App Quality (0-5 scale)

**5 — Excellent**
- App runs smoothly in browser with no errors
- Slot machine mechanic is clear and functional (spin, win/lose, tokens update)
- AI mockery is creative and present (humorous references, tone, or design)
- UI is polished and intuitive

**4 — Good**
- App runs with minor issues
- Slot machine works, tokens track correctly
- AI mockery is present but subtle
- UI is usable and reasonably designed

**3 — Acceptable**
- App runs but has minor bugs or awkward interactions
- Basic slot machine mechanic exists
- AI mockery is there but minimal or forced
- UI is functional but rough

**2 — Poor**
- App has significant bugs or incomplete features
- Slot machine mechanic is unclear or broken
- Little to no AI mockery
- UI is confusing or hard to use

**1 — Broken**
- App doesn't run or is unusable
- No clear slot machine mechanic
- No AI mockery
- No clear UI

**0 — Did not run**

---

## Code Quality (0-5 scale)

**5 — Excellent**
- Well-organized file structure (HTML/CSS/JS separated cleanly)
- Clear variable and function names
- Logic is easy to follow
- No redundant or messy code

**4 — Good**
- Organized structure with minor issues
- Generally clear naming conventions
- Logic is mostly readable
- Some minor redundancy

**3 — Acceptable**
- Basic organization, could be cleaner
- Mixed naming clarity
- Logic works but is convoluted in places
- Some unnecessary code

**2 — Poor**
- Disorganized or hard to follow
- Unclear variable/function names
- Logic is hard to trace
- Lots of redundancy or dead code

**1 — Bad**
- No clear structure
- Unintelligible code
- Logic is nearly impossible to follow

**0 — Unreadable**

---

## Scoring

- Rate each  on both scales independently
- Final rank = (App Quality + Code Quality) / 2
- Tiebreaker: App Quality wins if equal
